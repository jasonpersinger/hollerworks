# HOLLER.WORKS One-Pager

## Identity

- Display brand: `HOLLER.WORKS`
- Editorial/body use: `holler.works`
- Descriptor: `moderated tech and tech-adjacent jobs in Appalachia`

## Core Feel

- utilitarian but stylish
- grounded, regional, internet-native
- useful first, stylish second

## Voice

- direct
- human
- restrained
- lightly rough-edged

Avoid:

- buzzwords
- empty optimism
- generic “future of work” language

## Casing

- Lowercase works in branding, launch, social, and editorial copy.
- Use sentence case in UI, forms, and system copy.
- Avoid `Holler.Works`.

## Slash Device

- Use `//` as a branded header or opener.
- Use `>>` for a short supporting line when useful.

Examples:

- `// appalachia`
- `// launching holler.works`
- `>> stay. build. connected.`

## Palette

- Black: `#0E0E0E`
- Rust: `#F28927`
- Green: `#39803E`
- Gray: `#AAAAAA` / Light Gray: `#CCCCCC`
- rust leads, green supports, black is the canvas

## Logo + Visuals

- Keep the pixel mascot crisp and pixelated.
- Use the standardized wordmark asset when the full lockup is too tall.
- Prefer black backgrounds.
- Rust wordmark + green supporting line is the primary treatment.
- Monospace is core to the brand voice.

Avoid:

- smoothing pixel art
- random recolors
- glossy SaaS polish

## UI Rules

Keep:

- rails
- borders
- visible structure
- obvious actions and states

Avoid:

- rounded, soft UI language
- oversized whitespace
- generic corporate visuals

## Litmus Test

1. Does it look like HOLLER.WORKS at a glance?
2. Does it keep the black/rust/green system?
3. Does it sound human?
4. Does the old-web influence feel intentional?
5. Would it still feel credible to an employer or job seeker?
